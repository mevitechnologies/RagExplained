What is RAG?

RAG stands for Retrieval-Augmented Generation.

It is a technique that combines Information Retrieval and Large Language Models.

Instead of relying only on the knowledge stored inside an LLM, RAG allows AI systems to retrieve information from external documents before generating answers.

Think of RAG as an Open Book Exam.

The AI can search documents, retrieve relevant information, and then answer the user's question.

  • Retrieval + Generation Phase (Online)

📄 What is Chunking?

Imagine uploading a 500-page PDF into a RAG system.

Sending the entire PDF to the LLM is impossible because LLMs have context window limitations.

Therefore we split large documents into smaller pieces.

These smaller pieces are called Chunks.

Chunking is one of the most important steps in RAG.

Why Do We Need Chunking?

  • Reduces token usage
  • Improves retrieval accuracy
  • Makes search faster
  • Improves response quality
  • Helps vector databases work efficiently

🧠 What are Embeddings?

Computers do not understand words.

Computers understand numbers.

Embeddings convert text into numerical vectors.

These vectors capture the meaning of the text.

Example 


Dog

↓

[0.23, 0.54, 0.11, ...]

Cat

↓

[0.21, 0.57, 0.13, ...]

Notice that Dog and Cat produce similar vectors.

This is because they have similar meanings.

Why Embeddings Matter

  • Enable semantic search
  • Enable similarity matching
  • Improve retrieval accuracy
  • Power vector databases
  • Help understand context

🗄️ What is a Vector Database?

After creating embeddings, we need a place to store them.

A Vector Database stores embeddings instead of normal text.

It allows fast similarity search.

Traditional Database

Stores rows and columns.

Vector Database

Stores numerical vectors.

Database Description Best For
ChromaDB Simple local vector database Learning & Projects
FAISS Facebook similarity search engine Fast local search
Pinecone Managed cloud vector database Production systems
Qdrant Open-source vector search Enterprise projects
Weaviate AI-native vector database Large-scale RAG

🎯 What is Fine Tuning?

Fine Tuning means taking a pretrained model and training it further on your own dataset.

Example:

  • Medical Reports
  • Legal Documents
  • College Notes
  • Company Data

The model learns patterns from your specific domain.

Feature RAG Fine Tuning
